Did you use `=` or `==` or `===` to compare passwords (or something else)? Why? Would any of the others work?

I used "==" because we're looking for basic equality in this situation. We're not necessarily checking the "type" of the user's input. That said, using "===" would work in this case (I double-checked to make sure) because the input from the user is being converted into a string and the variable we are setting is also a string; they are the same types of objects.

Why is it a bad idea to store a password (or other sensitive information) in plain text in a Javascript file?

I think the main reason this would be a bad idea, is because this is all happening in the web browser. Meaning, anyone can look at the source code and figure out the password if they know where to look. Also, things would become very complicated very quickly if we had more than one password and more than one user.
