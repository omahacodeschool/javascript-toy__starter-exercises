Does it matter what order your if-statements are in?

Yes.

When testing ranges, did you use &&, ||, or something else entirely (e.g. nested if-statements)? If you used &&, why is that a better choice than || (and vice-versa)?

When testing ranges, I used the && operator. This operator is better than || because it will not evaluate true unless both conditionals pass whereas with || if either of the conditionals are true then it will evaluate to true.

What types of input produce a value of NaN after parsing? Is it important to check for NaN values? Is it possible for a NaN value to trigger any of the other if-statements? If so, how and why?

Any non-integer will produce a value of NaN after parsing. I don't think there's a reason to check for NaN values since the parseInt() serves as a pseudo check by producing either an integer value which can be processed correctly or NaN. I can not think of a time when a NaN value would trigger any of the other if-statments. The only one I could think of would be the zero range if somehow NaN is also evaluated as 0 but that didn't occur during my testing so I don't know if it's possible.
