If you don't parse and you use the == operator then yes, javascript does coerce the string into an integer and the secret number still works. Thats not the case with ===. Not sure exactly which is better, I guess whoever makes the site just needs to be aware of the difference.

Same as above, if you parse the user input into an integer than either == or === will always work as long as you are careful to store you "secret number" as an integer too.
