If you collect the user's input as a string (i.e. without using `parseInt()` to turn it into an integer), does `==` work to compare the input to the stored secret number? Would `===` work? Which is best?

'==' is a lenient comparison operator whereas '===' is a strict one. So, if you didn't convert the string to an integer, it wouldn't recognize 2 as being the same as "2" for example. So, if you wanted to spare yourself a line of code you might use the '==' but if you wanted to make sure that you were matching a password and wanted the match to be exact in all respects, you'd want to convert to an integer and use the '==="

If you collect the user's input as an integer (i.e. by using `parseInt()` to turn it into an integer), can you use `==` to compare? `===`? Which is better in this case?

If you convert it to an integer, you could definitely use '==". However, I'm not sure, in this instance, that there's really a better or worse to be found with this.

What is the difference between `==` and `===`?

As mentioned before '==" is a lenient comparison operator, meaning that when comparing an integer and a string, it'll convert them both to a common type and compare their value. The '===' is a strict comparison, so both sides of the operator have to be exactly the same.
