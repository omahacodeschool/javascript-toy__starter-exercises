What is the data-type (e.g. string or integer) of the information that comes in? What happens if the user inputs a number -- does it stay an integer? Use `typeof` and `console.log()` to help with this.

The data type of the information provided by prompt is either null or a string. As such, when the user inputs a number, it outputs as a string. So, if you wanted to do something with the number as a number, you'd need to convert it to either a float or an integer before doing that. It seems though like JS will, in some circumstances, if the string looks like it contains a number, do the conversion by itself behind the scenes. I'm not entirely sure how this happens, but heard it referenced when I was looking through documentation.

In these exercises, we use `prompt()` to collect information, but most websites do that ask for information do not use `prompt()` -- how do they collect information (what HTML elements do they use)? What might be an advantage to doing it that way instead of with `prompt()`? Are there any disadvantages? Why might you use `prompt()`?

We've already had experience in using the method you're most likely to see, i.e., HTML forms. Their are any number of advantages to doing it that way as opposed to doing it with a prompt. Among them, you have more control over the data type of the information passed, you can gather a greater variety of information within a single form, and a fair number of people out there simply find popups annoying. As for disadvantages, I'm not sure, I suppose that if you placed the form badly, it might go unnoticed and thus not get filled out. So, that might, if anything be an advantage of using a prompt, because it is actually prompting the user to enter information, it could be used to request information that's really vital to continuing on the website. Still, the fact that you don't see a ton of prompts out there today while wandering around the web seems to speak to a relative lack of utility in comparison with forms.
