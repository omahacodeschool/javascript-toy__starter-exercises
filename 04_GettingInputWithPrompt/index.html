  1. What is the data-type (e.g. string or integer) of the information that comes in? What happens if the user inputs a number.
    • The data-type of information that comes in is String. If a user inputs an Integer when a String was expected, the alert will return a String (just a String of the number entered). If the user inputs a String when an Integer is expected, the alert still returns a String.
  2. What happens if the user clicks "OK" (or equivalent) without typing anything in? What information gets assigned to a variable, if any?
    • Alert() just displays an empty String, console.log verifies this.
  3. How did you choose your variable names? If you gave a list of variable names to someone who hasn't seen your code, could they reasonably predict what information you might store in each?
    • Variable names were chosen based on the variable data they would be storing. If I gave a list of variable names to someone who hasn't seen my code they could reasonably predict what information we would want to store in each.
  4. If you completely changed a question you were asking the user, would it make sense to totally rename the variable? What if you changed the order of your questions?
    • If I completely changed a question I was asking the user it would make sense to rename the variable for the sake of maintainable code. It would not make sense to rename the variables if simply changing the order of the questions.
  5. In these exercises, we use prompt() to collect information, but most websites do that ask for information do not use prompt() -- how do they collect information (what HTML elements do they use)? What might be an advantage to doing it that way instead of with prompt()? Are there any disadvantages? Why might you use prompt()?
    • Most websites use HTML forms and POST the user input to their database. An advantage might be that the user's information is not stored on their browser vs. with prompt(), and hence more secure.
    • A disadvantage to using HTML forms and POST might be that with prompt(), we are assured to obtain some form of user input before the user can continue with their experience. I don't see that as being much of and advantage for prompt() though.
