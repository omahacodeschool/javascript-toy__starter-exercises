What's the difference between alert() and console.log()?

An 'alert()' opens a prompt box with which the user can interact where 'console.log()' prints lines directly to the console, where the user can't see. I found my 'console.log()' answers (and where they could be found locally) in Chrome by going to View >> Developer >> JavaScript Console. I imagine this is used for developers to leave notes for themself or other developers.

What's one situation where you might use an `alert()` to tell the user something?

One situation in which I might make use of an 'alert()' would be if a user attempted to leave a page in the middle of filling out a form without submitting it.

The prompt appearing in the 'alert()' might read something like this: "You haven't yet finished this form. Are you sure you want to leave this page?"

What's one way that `alert()` might get obnoxious?

An 'alert()' could get obnoxious if overused or used in an unwarranted situation. It feels a bit jarring as a user if unexpected and could create a negative relationship between the user and the site if not implemented wisely.
