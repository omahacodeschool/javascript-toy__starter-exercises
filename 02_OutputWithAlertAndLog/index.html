What's the difference between alert() and console.log()?

Alert pops up an alert box in front of the screen which you have to click "OK" to make go away. Console.log, I've read more than once, is "just like puts" in Ruby. I don't think this is the case. To my knowledge, "puts" doesn't put anything in a special log that's invisible to users.

What's one situation where you might use an `alert()` to tell the user something?

If someone's time-limited login session was about to expire due to inactivity on the page.

What's one way that `alert()` might get obnoxious?

It's obnoxious because you can't do anything else on the page until you click OK and make it go away.
