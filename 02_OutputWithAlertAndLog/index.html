What's the difference between alert() and console.log()?

An alert() is a public message with a pop-up, a console.log() is a message that one has to access the console to view. An alert() is forced upon you, whereas a console.log() has to be sought out.

What's one situation where you might use an `alert()` to tell the user something?

Well, if the name is a good indication, you should use it as an ALERT--it makes sure that the user acknowledges the warning or message before they can go forward in any way.

So, using some of our previous projects as examples, I could have used an alert() to let the admins of the swimming competition know that events were locked, or remind them before they even view the "delete swimmer" page that deleting a swimmer removes them from events, things like that.

What's one way that `alert()` might get obnoxious?

WELL, too many of them and you might make the user think they went to a spam page. They should be used only when information NEEDS to be brought front and center and even then, used sparingly. There is a flow to web browsing and disrupting that flow too much could make people avoid your work entirely.
