What's the difference between alert() and console.log()?

alert() pops up, then the page's HTML loads. From W3C -- The alert() method displays an alert box with a specified message and an OK button. An alert box is often used if you want to make sure information comes through to the user.

console.log() displays data via writing into the browser console.


What's one situation where you might use an alert() to tell the user something?

When you want to stop the user from proceeding until they acknowledge a message.


What's one way that alert() might get obnoxious?

Could be overused and draw the user's focus from the web app to the alert message.


Would it make sense to use console.log() to tell the user things? Why or why not?

No, because I don't think the user can view the console.log unless they do a CMD+alt+J or similar shortcut to view the page's javascript.


What's one way that console.log() might be a good tool for you to use?

For debugging, or possibly tracking user interactions with your web app.
