What's the difference between alert() and console.log()?

Simply put, an alert() is displayed directly to the user whereas a console.log() is stored to the console and can only be viewed on the console. Also, the alert() is geared more towards giving the user information while the console.log is set up more to give the developer information and keep track of what javascript is doing behind the scenes.

What's one situation where you might use an `alert()` to tell the user something?

Well, as we use alerts in this section, they can be used to provide urgen information to the user(though, how saying hi and introducing myself doesn't really meet that threshold). So, if you wanted to tell them to be sure to do something while on the page to ensure they get full functionality out of it. That might be one thing. You might also use the alert's popup to convey that certain tasks have been completed.

What's one way that `alert()` might get obnoxious?

Well, anything that's poping up and needs to be closed before you can do anything else is liable to get annoying or obnoxious if overused. For example, on our travel app, if you set up an alert to tell the user every time they'd successfully filled out one form and moved on to the next cell that might entail them having to click through a dozen popups or more.

Would it make sense to use `console.log()` to tell the user things? Why or why not?

It would probably not make sense to use console.log() to tell the user things because it's not displayed directly to the user. They'd need to know how to access the console to see what was put there via console.log().

What's one way that `console.log()` might be a good tool for you to use?

Well, you could track javascript activity on the page and not clutter up the user's screen with information they have little interest in. It would give you a way to track what javascript is up to.

In addition, googling gave me these other good reasons to use console log with debugging as opposed to trying to do it with alerts.

  • alert() is blocking
  • alert() cannot be easily suppressed in non-debug environment
  • console typically formats your objects nicely and allows to traverse them
  • logging statements often have an interactive pointer to code which issued logging statement
  • you cannot look at more than one alert() message at a time
  • consoles can have different logging levels with intuitive formatting
