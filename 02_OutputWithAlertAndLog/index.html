What's the difference between alert() and console.log()?

The alert() function brings up a message box that is intended for the user to see (and the user must click "ok" to dismiss the message). The console log is not for the user to see. It is similar to "puts" in Ruby.

What's one situation where you might use an `alert()` to tell the user something?

Alerts probably shouldn't be used all that often. I would think they should only be used when absolutely necessaary. In general, they should be used in scenarios where it is vital that the user is given some information or warning.

A more specific example of when you might want to use an alert would be as follows: Say we have a website with a shopping cart and in order for the user's selections to be saved, they most "save" or "submit" the selected items first. In this example, if we had a user that selected several items but never clicked "save" and then tried to close the browser window, we might want to use an alert to notify the user that any selections they have made will be lost upon closing, if they do not click the "save" button.

Would it make sense to use console.log() to tell the user things? Why or why not?

No. Definitely not. Probably the main reason - to which I made reference above - is that the console is not designed to be viewed by the user (unless that user happens to be a dev... but I digress). The console is for developers and is a helpful tool for detecting bugs, confirming that things are working correctly and so on.

What's one way that console.log() might be a good tool for you to use

As I stated above, the console is great for debugging. If somewhere in our javascript we have a function that is supposed to do something when a button is clicked, and let's say nothing was happening, we could add a console.log() statement into our code to inform us if that button was clicked. If the console does not show the console.log statement, this might tell us that the button was never clicked in the first place.
