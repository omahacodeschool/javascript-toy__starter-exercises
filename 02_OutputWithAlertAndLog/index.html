What's the difference between alert() and console.log()?

An alert opens a 'popup window' to give a message to the user, the console.log() provides an output on the console.

What's one situation where you might use an `alert()` to tell the user something?

Possible greetings, to give important information, or specifically notify the user in a more direct way than just printing the information on the screen.

What's one way that `alert()` might get obnoxious?

Over use of altert()s could deffinately get annoying, and really I think should be saved for times when it could be important or particularly relevant.

Would it make sense to use `console.log()` to tell the user things? Why or why not?

The console.log() is really more to see what the computer is doing as a method to track that. Seeing as though only the end user is privvy to this information, it doesnt make it a primary means for displaying information to the user.

What's one way that `console.log()` might be a good tool for you to use?
