1. Why don't you need to use the `var` keyword every time you use a variable (right now, we only use it when we create the variable)? What does `var` do?

You don't need to use 'var' every time you use a variable because you only do so when creating a variable. Unlike Ruby, which seems to know that the thing to the left of the equal sign is a variable, in javascript, 'var' is the equivalent of going "Okay computer, tell my program that we're going to make a variable now." After the initial creation, the program knows that the collection of characters you used is now a variable. and calling 'var' again would be overkill.

2. What happens if you try to do output with a variable without first assigning a value to that variable? What happens? Does the program still work as intended? Do error messages show up in the console? If so, what error message?

If you attempt to output a variable like 'var test;' and then call it in an alert, it will return a value of 'undefined' when called. For me, the program still seemed to run as intended. I didn't get any error messages in console, either. However, I wasn't trying to force that variable to interact with another variable (like a + b), which I am sure would lead to some issues.
